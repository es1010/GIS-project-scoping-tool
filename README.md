**GeoScope**

GeoScope is a lightweight GIS project scoping tool published with GitHub Pages. It helps users turn a project idea into a structured project brief through an interactive form instead of a static template. Built with assistance from AI tools.

The tool supports GIS-specific planning by guiding users through project goals, methodology, expected outputs, data choices, risks, and approvals. It also helps users compare free/open and paid/licensed data sources and generate editable template text for project planning.

**Features**

·        Interactive GIS project scoping form.

·        Pre-filled methodology workflows that users can edit.

·        Multiple-choice expected outputs.

·        Risk register with custom risk fields for likelihood, impact, and mitigation.

·        Data source selection with free/open and paid options.

·        Copy-to-document and download-as-text export options.

**Use cases**

GeoScope can be used for:

·        Research GIS projects.

·        Client or consulting GIS work.

·        NGO or community mapping projects.

·        Internal planning for spatial analysis or web mapping tasks.

·        Participatory GIS (PGIS) workflows.

**How it works**

1.      Enter the project title and goal.

2.     Select a methodology and edit the suggested step-by-step workflow if needed.

3.      Choose expected outputs, data options, and approval roles.

4.     Review risks, including optional custom risks.

5.      Copy or download the generated project template text.

**GitHub Pages deployment**

To publish the tool with GitHub Pages:

1.      Create a GitHub repository.

2.     Upload the HTML file as the main page.

3.      Rename the file to index.html if needed.

4.     In the repository settings, open **Pages**.

5.      Set the source to the main branch and root folder.

6.     Save the settings and wait for GitHub Pages to publish the site.

**Project goal**

The aim of GeoScope is to make GIS project planning faster, clearer, and more adaptable by giving users a practical starting structure they can modify rather than forcing them to begin from a blank document.
